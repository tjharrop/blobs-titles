Notes 📝


Anatomy of a full-stack component

Traditional Design System

UI Design toolkit

  • Look & feel
  • Digital UI (HTML & CSS)

UI Dev toolkit

  • The code for a view-layer component (Vue/React)
  • API integration (JS)

Full-stack Design System

Tech toolkit

  • App boilerplate
  • Reliable and secure IaaS & PaaS

Behind-the-scenes user-centricity

  • Content guidance
  • Helping users come unstuck
  • Support channels
  • Approachability & accessibility
Notes 📝

Making DesignOps 2.0 + Full Stack Design Systems work

Developers

Need components with no unanswered questions, where they might have to interpret intent

UI/Product people

Need an accessible tool to communicate what to build, which creates fully usable prototypes

UX/Content people

Need contextual patterns, and a platform to share and extend them

Teams

Need to use the same components, from a single source of truth

Power up 1: Creative technologists

Creative technologists are people who have a creative mindset and a technical skillset.

They're most often in development roles, as the marketable skills are so similar.

A creative technologist is a UXer

Notes 📝

Power up 2: Contextual patterns

What the user is trying to do, not what they're using to do it.

Channel agnostic.

Based on a micro-goal.

Allow linkage between products, business logic, content, and support.
