Anatomy of a full-stack component

Traditional Design System

UI Design toolkit

  • Look & feel
  • Digital UI (HTML & CSS)

UI Dev toolkit

  • The code for a view-layer component (Vue/React)
  • API integration (JS)

Full-stack Design System

Tech toolkit

  • App boilerplate
  • Reliable and secure IaaS & PaaS

Behind-the-scenes user-centricity

  • Content guidance
  • Helping users come unstuck
  • Support channels
  • Approachability & accessibility

Making DesignOps 2.0 + Full Stack Design Systems work

Developers

Need components with no unanswered questions, where they might have to interpret intent

UX/Content people

Need contextual patterns, and a platform to share and extend them

Teams

Need to use the same components, from a single source of truth
